# Experiment Report: Avatar-World Transfer

## Setup

This experiment tests whether a higher controller can retain coherent organisation across different world modes when the body substrate is preserved.

### Environments
- **Native physical world**: `NativePhysicalWorld` — forward-progress + stability reward
- **Simplified embodied world**: `SimplifiedEmbodiedWorld` — target-approach with distance reward
- **Avatar remapped world**: `AvatarRemappedWorld` — abstract 2D navigation with stability-modulated movement

### Body
- `FlyBodyLayer` with default config (same across all worlds)

### Episode parameters
- **Episode length**: 64 steps
- **Seeds**: 5 (0–4)

## Hypotheses

1. **H1**: Controllers with memory show more consistent metric profiles across world modes than reactive controllers.
2. **H2**: The body substrate provides a common foundation that enables some cross-world invariants (e.g., stability metrics) even for reactive controllers.
3. **H3**: Task performance is world-specific (different reward structures), but integration markers (state persistence, self/world separation) should transfer if the controller has genuine internal organisation.
4. **H4**: The random controller shows uniformly low performance across worlds (negative control).

## Methods

### Controllers tested
- Reduced descending controller
- Memory controller
- Planner controller
- Random controller (negative control)

### Cross-world comparison
For each controller, compare metric profiles across the three world modes:
1. **Within-controller cross-world variance**: How much do metrics change when the world changes but the controller stays the same?
2. **Within-world cross-controller variance**: How much do metrics change when the controller changes but the world stays the same?

If within-controller variance is LOW (metrics are stable across worlds), the controller has world-agnostic properties.

All core metrics plus target representation stability where applicable.

## Caveats

- The three world modes differ substantially in reward structure and dynamics — direct metric comparison requires normalisation.
- The avatar world uses stability-modulated movement which couples the body substrate to world dynamics in a specific way — not a clean ablation.
- Same body substrate does not mean same effective dynamics — the body interacts differently with each world.
- Short episodes may not reveal long-horizon transfer properties.

## Failure cases

- If all controllers perform equally across worlds, world remapping is trivially easy and doesn't test anything meaningful.
- If NO controller transfers well (all show high cross-world variance), the architecture doesn't support world-agnostic control and the three-layer separation needs revision.
- If the random controller performs comparably to structured controllers in the avatar world, the avatar world is too simple.

## Next steps

1. Train controllers in one world, test zero-shot in another to measure genuine transfer.
2. Measure representation similarity across worlds using CKA or CCA on internal states.
3. Add more diverse world modes with distinct physics and dynamics.
4. Use the `FlyDualWorldEnv` for runtime world-switching experiments.
5. Quantify the body substrate's contribution to cross-world invariance by comparing embodied vs bodyless transfer.
