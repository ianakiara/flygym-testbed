# Experiment Report: Controller Interoperability

## Setup

This experiment swaps controller families over the same body/world to identify what latent or behavioural invariants survive translation across controllers.

### Environment
- **World**: `SimplifiedEmbodiedWorld` (target-approach with distance reward)
- **Body**: `FlyBodyLayer` with default config
- **Episode length**: 64 steps
- **Seeds**: 5 (0–4)

## Hypotheses

1. **H1**: Controllers sharing ascending feedback produce more similar latent state trajectories (higher correlation) than controllers ignoring feedback.
2. **H2**: Body-mediated invariants (stability trajectory, contact patterns) are more consistent across controllers than action-space invariants.
3. **H3**: The random controller shows low interoperability with all others, serving as a negative control.
4. **H4**: Memory and planner controllers show convergent representations because they both exploit temporal structure.

## Methods

### Controllers tested
- Reduced descending (standard reduced interface)
- Memory controller (explicit memory buffer + recurrent hidden state)
- Planner controller (subgoal decomposition)
- No-ascending-feedback controller (descending only)
- Reflex-only controller (no higher-level control)
- Random controller (sanity check)

### Interoperability metrics
- **Action distribution similarity**: Mean, std, range of move/turn intents across controllers
- **Latent state correlation**: Correlation of ascending summary feature trajectories (e.g., stability)
- **Reward trajectory correlation**: Correlation of reward time series across controllers
- **Composite interoperability score**: Average of reward and latent correlations

### Comparison design
All 15 pairwise comparisons between 6 controllers on the same body/world.

## Baselines

| Controller | Role |
| --- | --- |
| Reduced descending | Reference condition |
| Random | Negative control (should show low interoperability) |
| No-ascending | Tests whether ascending feedback drives shared representations |

## Caveats

- Interoperability is measured via ascending summary features, not true internal latent states — this is a proxy.
- Heuristic controllers produce limited behavioural diversity; trained controllers would provide richer comparisons.
- Short episodes may not allow convergent dynamics to emerge.
- Same body/world means the body substrate dominates — some "interoperability" may be body-driven rather than controller-driven.

## Failure cases

- If all controllers show identical metric profiles, the body/world dominates and controllers are interchangeable — suggesting the architecture trivialises the controller's role.
- If no controller pair shows high interoperability, all representations are entirely controller-specific with no shared structure.
- If the random controller shows high interoperability, the metric is capturing environmental regularities rather than controller properties.

## Next steps

1. Use trained neural controllers for richer representation comparison.
2. Compute CKA (Centered Kernel Alignment) or CCA between internal state trajectories.
3. Test interoperability across different world modes (cross-world × cross-controller).
4. Measure transfer learning success: pre-train in one controller, fine-tune in another.
