# Experiment Report: History Dependence

## Setup

This experiment creates scenarios to test whether history is a necessary state variable for the controllers, using advanced persistence metrics.

### Environment
- **World**: `SimplifiedEmbodiedWorld`
- **Body**: `FlyBodyLayer` with default config
- **Episode length**: 64 steps
- **Seeds**: 5 (0–4)

## Hypotheses

1. **H1**: Controllers with explicit memory show higher cross-time mutual information than memoryless controllers.
2. **H2**: State decay curves are slower (higher half-life) for memory-augmented controllers.
3. **H3**: Predictive utility of past hidden state is higher for memory controllers — past state predicts future observations.
4. **H4**: Hysteresis scores are highest for memory controllers — same current state leads to different actions depending on history.
5. **H5**: The random controller shows near-zero history dependence on all metrics (negative control).

## Methods

### Controllers tested
- **Reduced descending**: Has a scalar `memory_trace` — minimal history.
- **Memory controller**: Explicit memory buffer + recurrent hidden state — rich history.
- **Random**: No history at all — negative control.

### Metrics
| Metric | What it measures |
| --- | --- |
| Cross-time MI (lags 1–5) | Mutual information between feature values at different temporal lags |
| State decay curve (lags 1–10) | Autocorrelation at increasing lags — how fast state information decays |
| Decay half-life | Lag at which autocorrelation first drops below 0.5 |
| Predictive utility | Correlation between past state and future observations (horizon=3) |
| Hysteresis score | Action variance conditioned on state trajectory direction (rising vs falling) |
| History dependence | Action variance within coarse-state buckets (from core metrics) |

### Design
All three controllers run on identical episodes (same seeds, same environment). Differences in metrics are attributable to controller architecture, not environmental variation.

## Baselines

| Controller | Role |
| --- | --- |
| Random | Negative control — should show zero history dependence |
| Reduced descending | Minimal memory baseline (scalar trace only) |
| Memory | Full memory condition |

## Caveats

- MI estimation via histograms is coarse for short time series (64 steps) — results should be interpreted as lower bounds.
- Hysteresis depends on state binning resolution — different bin counts may yield different scores.
- Heuristic controllers have simple, deterministic memory — not comparable to learned representations (e.g., LSTM hidden states).
- The "matched-current-state" ideal is not perfectly achieved — the HistoryDependenceTask provides a better controlled comparison.
- Predictive utility uses linear correlation — nonlinear predictive relationships would be missed.

## Failure cases

- If the memory controller shows no more history dependence than the reduced descending controller, the explicit memory buffer adds no value beyond the scalar trace.
- If the random controller shows non-zero history dependence, the metric is capturing environmental autocorrelation rather than controller memory.
- If all controllers show identical decay curves, the environment dominates state dynamics and controller architecture is irrelevant.

## Next steps

1. Test with trained RNN controllers (LSTM/GRU) for genuine learned persistence.
2. Use longer episodes (256+ steps) to improve MI estimation quality.
3. Create explicitly matched-state scenarios using the `HistoryDependenceTask` (visit waypoints in different orders from same starting state).
4. Add nonlinear predictive utility measures (e.g., mutual information instead of correlation).
5. Separate controller-internal history from environment-induced history via shuffle controls.
