# Experiment Report: Ascending Feedback Ablation

## Setup

This experiment systematically disables ascending feedback channel groups to determine which channels are necessary for stable control and self/world disambiguation.

### Environment
- **World**: `SimplifiedEmbodiedWorld`
- **Body**: `FlyBodyLayer` with varying `disabled_feedback_channels`
- **Episode length**: 64 steps
- **Seeds**: 5 (0–4)

## Hypotheses

1. **H1**: Stability feedback is the most critical channel for coherent higher-level control — ablating it causes the largest degradation.
2. **H2**: Locomotion feedback is necessary for self/world separation — without it, the controller cannot distinguish self-induced motion from external perturbation.
3. **H3**: Target feedback ablation degrades task performance but NOT stability or persistence metrics — it is task-specific rather than architecturally critical.
4. **H4**: Ablating ALL channels collapses behaviour to random-like performance, confirming ascending feedback is collectively necessary.
5. **H5**: Graded ablation (removing channels one at a time vs. in groups) reveals a channel importance hierarchy.

## Methods

### Ablation conditions
| Condition | Channels disabled |
| --- | --- |
| Baseline (no ablation) | None |
| Stability off | `{stability}` |
| Locomotion off | `{locomotion}` |
| Contact off | `{contact}` |
| Effort off | `{effort}` |
| Target off | `{target}` |
| Stability + locomotion off | `{stability, locomotion}` |
| Stability + locomotion + contact off | `{stability, locomotion, contact}` |
| All channels off | `{stability, locomotion, contact, effort, target}` |

### Controllers tested
- Reduced descending controller
- Memory controller

### Metrics
All core metrics: task performance, state persistence, history dependence, self/world separation, stabilisation quality, seam fragility.

## Baselines

| Condition | Role |
| --- | --- |
| No ablation | Positive baseline (full feedback) |
| All channels off | Ceiling ablation (maximum degradation) |
| Random controller (from Phase 6) | Reference for "no useful control" performance |

## Caveats

- Ablation zeros feature values rather than removing them from the observation — controllers may learn to treat zeros as a signal.
- Channel groups are coarse (e.g., "stability" includes thorax height and body speed) — individual features within a group may vary in importance.
- Only two controller types tested — results may not generalise to trained controllers that have learned specific channel dependencies.
- The ascending adapter computes derived features — ablating raw channels may have different effects than ablating summary features.

## Failure cases

- If no single-channel ablation degrades performance significantly, channels are redundant — the controller can compensate using remaining channels.
- If all single-channel ablations equally degrade performance, no channel hierarchy exists — all are equally important.
- If the memory controller is immune to ablation but the reduced-descending controller is not, memory compensates for missing feedback — which is interesting but confounds the channel importance assessment.

## Next steps

1. Fine-grained per-feature ablation within each channel group.
2. Noise injection (Gaussian perturbation) instead of zeroing — test partial degradation rather than binary on/off.
3. Test with trained controllers that have learned to depend on specific channels.
4. Cross-reference with global disruption signature metrics to assess integration.
5. Add temporal ablation (channels disabled at specific time windows) to test dynamic dependency.
